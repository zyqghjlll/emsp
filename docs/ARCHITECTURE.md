# EMSP DDD架构设计文档

## 项目概述

EMSP (Electric Mobility Service Provider) 是一个基于领域驱动设计(DDD)的电动汽车充电服务管理系统。本项目展示了如何在企业级Java项目中应用DDD架构模式。

## 架构设计原则

### 1. 分层架构
```
┌─────────────────────────────────────┐
│              API Layer              │  ← 接口层：REST API、消息队列
├─────────────────────────────────────┤
│           Application Layer         │  ← 应用层：业务用例编排
├─────────────────────────────────────┤
│             Domain Layer            │  ← 领域层：核心业务模型
├─────────────────────────────────────┤
│         Infrastructure Layer        │  ← 基础设施层：技术实现
└─────────────────────────────────────┘
```

### 2. DDD核心概念实现

#### 聚合根 (Aggregate Root)
- **Location**: 充电站点聚合根
- **Evse**: 充电设备聚合根

#### 实体 (Entity)
- **Connector**: 充电接口实体

#### 值对象 (Value Object)
- **LocationId**: 站点标识
- **EvseId**: 设备标识
- **Coordinate**: 坐标值对象
- **BusinessHours**: 营业时间值对象

#### 领域服务 (Domain Service)
- **LocationDomainService**: 站点领域服务
- **EvseDomainService**: 设备领域服务

#### 仓储 (Repository)
- **LocationCmdRepository**: 站点命令仓储
- **EvseCmdRepository**: 设备命令仓储

### 3. CQRS模式实现

#### 命令端 (Command Side)
- 处理写操作
- 使用命令对象封装业务操作
- 发布领域事件

#### 查询端 (Query Side)
- 处理读操作
- 使用专门的查询模型
- 支持分页查询

### 4. 事件驱动架构

#### 领域事件
- **LocationChangedEvent**: 站点变更事件
- **EvseChangedEvent**: 设备变更事件
- **LocationEvseChangedEvent**: 站点设备关联变更事件

#### 事件发布
- 使用Spring事件机制
- 支持异步事件处理
- 事件持久化存储

## 技术实现细节

### 1. 依赖注入
- 使用Spring的依赖注入
- 接口与实现分离
- 支持Mock测试

### 2. 事务管理
- 使用Spring事务注解
- 保证数据一致性

### 3. 异常处理

### 4. 数据验证
- 使用Hibernate Validator
- 参数校验与业务校验分离
- 自定义验证规则
